# 11.2 Operating Retrieval And Memory

_Page Type: Chapter Index | Maturity: Review-Ready_

This section turns retrieval design into operating review. Use it when teams need to prove that freshness, permissions, retention, and fallback behavior still hold once retrieval touches real sources, users, and rollout pressure.

## Section Map

- 11.2.1 [Worked Retrieval Scenarios](11-02-01-worked-retrieval-scenarios.md)
- 11.2.2 [Patterns And Anti-Patterns](11-02-02-patterns-and-anti-patterns.md)

## What This Section Helps Decide

- which operating posture fits the dominant knowledge shape, access model, and source volatility
- which recurring retrieval patterns are healthy reuse versus hidden persistence, stale state, or permission leakage
- which evidence outputs should exist before retrieval or memory is treated as production-ready
- which rollout changes should reopen privacy, security, oversight, or adaptation review before wider deployment
