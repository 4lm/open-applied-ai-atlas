# 1. Scope And Principles

_Page Type: Chapter Index | Maturity: Review-Ready_

This chapter turns the atlas mission into practical boundaries for reading, contribution, and later decision work. Use it when the open question is still whether a topic belongs in the atlas, which roles need to align first, or which cross-cutting priorities should shape every later comparison.

Without this front door, later chapters are too easy to read as disconnected tool surveys, generic GenAI commentary, or local implementation advice that quietly drops openness, sovereignty, portability, privacy, compliance, or lock-in from view.

## How To Read This Chapter

- Start with [1.1 Mission And Scope](01-01-00-mission-and-scope.md) when the team still needs a stable definition of what the atlas covers, who it serves, and which organizational contexts and AI/ML system families belong in frame.
- Move to [1.2 Using The Atlas](01-02-00-using-the-atlas.md) when orientation now needs to become a role-based reading path, a coordination pattern, or a worked journey across later chapters.
- Use [1.3 Repository Contract](01-03-00-repository-contract.md) when the issue is no longer topical scope alone, but how repository-level guidance, numbered chapter content, and contribution boundaries should stay separated.

## What This Chapter Helps Decide

- whether a topic belongs in the atlas at all or should be treated as out of scope, pure research, legal advice, or vendor marketing rather than implementation guidance
- which readers need to converge first before architecture, vendor, governance, or operating decisions fragment across functions
- which cross-cutting priorities must remain visible even when later chapters focus on models, gateways, sourcing, or standards
- when the unresolved problem has shifted into taxonomy, use-case classification, stack design, governance review, or repository maintenance rather than orientation
