# 1.2 Using The Atlas

_Page Type: Chapter Index | Maturity: Draft_

This section turns chapter `1` into a practical reading and coordination surface. Use it when the atlas already fits the problem, but the team still needs to decide where different readers should enter, how they should reconverge before downstream choices diverge, and what a sensible path through later chapters looks like for real organizational situations.

## Section Map

- 1.2.1 [Role-Based Reading Paths](01-02-01-role-based-reading-paths.md)
- 1.2.2 [Worked Reader Journeys](01-02-02-worked-reader-journeys.md)

## What This Section Helps Decide

- which chapter sequence best fits product scoping, architecture design, governance review, sourcing evaluation, or small-team delivery work without flattening those roles into one generic AI path
- which assumptions different readers must align on before they compare vendors, choose architectures, or widen rollout, especially around use-case class, control burden, data boundaries, and portability posture
- what a useful reading pass should produce before the team moves on, including a shared starting chapter set, explicit handoff points, and named triggers for reopening earlier assumptions
- when chapter `1` is still doing useful orientation work versus when the unresolved issue has shifted into taxonomy, stack design, governance, evaluation, oversight, or sourcing analysis

## Reading Boundaries

- Use [1.2.1 Role-Based Reading Paths](01-02-01-role-based-reading-paths.md) when the main problem is choosing a starting sequence for product, architecture, ML, assurance, sourcing, or leadership readers who need different entry points but still need to converge later.
- Use [1.2.2 Worked Reader Journeys](01-02-02-worked-reader-journeys.md) when the team needs scenario-led examples that show how those reading paths behave in public-sector, enterprise-platform, nonprofit, or review-heavy settings.
- Revisit [1.1 Mission And Scope](01-01-00-mission-and-scope.md) if the group still cannot agree on reader alignment, system family, organizational setting, or the cross-cutting priorities that should stay visible before it debates sequence.
