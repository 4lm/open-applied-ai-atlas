# 1.3 Repository Contract

_Page Type: Chapter Index | Maturity: Review-Ready_

This section defines the boundary between repository-level guidance and numbered topic content. Use it when the atlas mission already fits, but the open question is now where a rule, note, or change belongs so root guidance stays durable and topical chapters stay focused on applied AI and ML knowledge rather than local process copy.

## What This Section Helps Decide

- whether material belongs in root guidance, delivery tracking, or the numbered chapter system
- when a page should link to repository rules instead of restating them inside topical content
- which root documents define contribution workflow, editorial constraints, operator state, or public navigation before a proposed edit touches chapter content
- when the unresolved problem has shifted from repository structure into topic scope, reader sequencing, taxonomy, or implementation guidance

## Reading Boundaries

- Use [1.3.1 Root Docs And Topic Chapters](01-03-01-root-docs-and-topic-chapters.md) when the immediate problem is deciding whether a rule, reference, or explanation belongs at repository root or inside `docs/`.
- Revisit [1.1 Mission And Scope](01-01-00-mission-and-scope.md) if the group is still arguing about what the atlas covers, which readers it serves, or which cross-cutting priorities should remain in view before repository placement is discussed.
- Revisit [1.2 Using The Atlas](01-02-00-using-the-atlas.md) if the real issue is chapter sequence, role-based entry, or reader handoff rather than repository structure.
- Move to [README.md](../../README.md), [CONTRIBUTING.md](../../CONTRIBUTING.md), [EDITORIAL_RULES.md](../../EDITORIAL_RULES.md), or [.delivery/STATUS.md](../../.delivery/STATUS.md) when the next step is acting on public navigation, contribution workflow, editorial rules, or tracked delivery state rather than staying in chapter `1`.
