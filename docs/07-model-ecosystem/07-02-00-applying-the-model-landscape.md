# 7.2 Applying The Model Landscape

_Page Type: Chapter Index | Maturity: Draft_

This section turns model selection into operating review. Use it when a team already has a plausible model shortlist and now needs to prove that portfolio fit, openness posture, evaluation burden, refresh discipline, and sourcing consequences still hold under real delivery pressure.

## Section Map

- 7.2.1 [Worked Model Selection Scenarios](07-02-01-worked-model-selection-scenarios.md)
- 7.2.2 [Patterns And Anti-Patterns](07-02-02-patterns-and-anti-patterns.md)

## What This Section Helps Decide

- which selection posture fits internal assistants, extraction-heavy workflows, sovereignty-constrained deployments, and model-refresh decisions where capability, control, and operating burden pull in different directions
- which recurring operating shapes produce a governed model portfolio versus benchmark chasing, unmanaged provider sprawl, or hidden dependence on one opaque release stream
- which review outputs should exist before model use expands, including approved shortlist criteria, baseline evaluation packs, adaptation triggers, exception records, deprecation rules, and fallback paths
- which changes should reopen use-case framing, hosting, adaptation, evaluation, or sourcing review, including new modalities, larger context demands, self-hosting requirements, license changes, or abrupt shifts in supplier roadmap dependence
