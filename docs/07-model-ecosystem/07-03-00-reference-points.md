# 7.3 Reference Points

_Page Type: Chapter Index | Maturity: Draft_

This section gathers the named model catalogs, ecosystem platforms, vendors, and stewardship actors that matter after the model-selection frame is already clear. Use it to compare access surfaces and ecosystem influence without turning the chapter into a leaderboard or confusing organizations, model programs, hubs, and adjacent control layers as the same kind of decision.

## Section Map

- 7.3.1 [Tools And Platforms](07-03-01-tools-and-platforms.md)
- 7.3.2 [Vendors And Projects](07-03-02-vendors-and-projects.md)

## What This Section Helps Decide

- which model-access surfaces belong on the shortlist, including direct provider APIs, cloud model catalogs, open-weight programs, public hubs, and local packaging ecosystems
- whether the team's real dependency is on a model family, a platform control plane, a cloud estate, or an ecosystem steward whose roadmap and governance posture will shape later choices
- where openness, sovereignty, portability, lifecycle transparency, and concentration risk still change the answer even after the workload and method fit are understood
- when a named-tool question is really a hosting, gateway, adaptation, evaluation, or sourcing issue rather than a reference-section decision

## Reading Boundaries

- Use [7.3.1 Tools And Platforms](07-03-01-tools-and-platforms.md) when the team needs to compare managed model catalogs, open-model ecosystems, discovery hubs, and local access layers by stack role, openness posture, and control implications.
