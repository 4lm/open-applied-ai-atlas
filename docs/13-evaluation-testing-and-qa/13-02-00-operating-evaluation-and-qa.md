# 13.2 Operating Evaluation And QA

_Page Type: Chapter Index | Maturity: Review-Ready_

This section turns evaluation design into operating review. Use it when a team already knows the evidence types that matter and now needs to decide how scenario packs, release gates, ownership, and re-evaluation cadence work under real rollout pressure.

## Section Map

- 13.2.1 [Worked Evaluation Scenarios](13-02-01-worked-evaluation-scenarios.md)
- 13.2.2 [Patterns And Anti-Patterns](13-02-02-patterns-and-anti-patterns.md)

## What This Section Helps Decide

- which scenario mix should anchor the release packet for retrieval assistants, model-routing changes, action-capable agents, predictive services, and similar systems
- which recurring operating shapes produce reliable release evidence versus polished demos, narrow benchmark wins, or unowned reviewer judgment
- which changes should reopen evaluation before rollout expands, including model swaps, prompt and routing changes, tool additions, policy changes, or live incidents
- when the binding question has moved into monitoring, abuse resistance, oversight, or adaptation design rather than evaluation operations alone
