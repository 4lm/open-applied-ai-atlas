# 16.3.1 Standards And Bodies

This subsection holds the chapter's named comparison material. Read it as a reference layer, not as a substitute for the chapter's conceptual and implementation sections.

This file keeps formal standards, public bodies, and reference frameworks in one place so they can be compared without overwhelming the chapter front door.

## How To Use This File

- Frameworks solve different jobs and should not be treated as interchangeable.
- A standard is only useful when translated into design, control, or operating decisions.
- Public bodies and standards should not be scored like software platforms.

## Selection Heuristic

| If the oversight problem is mainly about... | Start with... |
| --- | --- |
| Board and executive accountability | ISO/IEC 38507, ISO/IEC 42001, COBIT |
| Working practices and run-state operations | ITIL and DORA research |
| Translating principles into tasks and evidence | NIST AI RMF Playbook and ISO/IEC 42005 |
| EU implementation readiness | EU AI Act, AI Pact, and chapter `04` |
