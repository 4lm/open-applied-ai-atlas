# 10.3 Reference Points

_Page Type: Chapter Index | Maturity: Draft_

This section gathers the named frameworks, workflow runtimes, and control artifacts that matter after the autonomy choice is already clear. Use it to compare orchestration options without confusing agent frameworks, durable workflow engines, and review packets as interchangeable answers to the same operating problem.

## Section Map

- 10.3.1 [Tools And Platforms](10-03-01-tools-and-platforms.md)
- 10.3.2 [Controls And Artifacts](10-03-02-controls-and-artifacts.md)

## What This Section Helps Decide

- which orchestration layer belongs on the shortlist: agent framework, workflow engine, or a narrower application integration that keeps tool use and state explicit
- whether the real dependency is on a coding framework, a managed tracing surface, a low-code automation platform, or a durable execution layer whose retry and approval semantics will shape later controls
- where openness, sovereignty, portability, and managed-platform dependence still change the answer even after the team agrees that some agent behavior is justified
- when a named-tool question is really a gateway, retrieval, evaluation, observability, security, oversight, or sourcing issue rather than a reference-section decision
