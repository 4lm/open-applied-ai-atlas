# 10.2 Operating Agentic Systems

_Page Type: Chapter Index | Maturity: Review-Ready_

This section turns agent design into operating review. Use it when a team already has a plausible autonomy choice and now needs to prove that approval paths, tool boundaries, trace coverage, rollback options, and workflow fallbacks still hold once the system touches real work.

## Section Map

- 10.2.1 [Worked Agentic Scenarios](10-02-01-worked-agentic-scenarios.md)
- 10.2.2 [Patterns And Anti-Patterns](10-02-02-patterns-and-anti-patterns.md)

## What This Section Helps Decide

- which operating posture fits content-only assistants, workflow-led automation, bounded action agents, and browser-capable pilots where flexibility, reversibility, and operator burden conflict
- which recurring operating shapes produce reviewable autonomy versus agent theater that hides unowned tools, missing action classes, weak rollback design, or no durable state
- which control outputs should exist before autonomy expands, including tool inventories, approval matrices, pilot-scope notes, disablement paths, trace locations, and incident playbooks
- which changes should reopen gateway, retrieval, evaluation, security, oversight, or sourcing review, including new tools, broader action classes, browser access, long-lived memory, multi-agent delegation, or privileged data exposure
