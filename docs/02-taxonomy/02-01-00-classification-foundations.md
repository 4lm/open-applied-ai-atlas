# 2.1 Classification Foundations

_Page Type: Chapter Index | Maturity: Draft_

This section sets the reusable distinctions that later chapters are expected to inherit rather than restate. Use it when a team is still deciding what kind of thing it is looking at, which labels are canonical, which cross-cutting dimensions actually belong in the analysis, and whether a comparison table should exist at all yet.

It is most useful for chapter authors, reviewers, architects, governance leads, and sourcing or procurement readers who need stable language before they trust a comparison, control note, or design summary. The same class-first discipline matters for generative systems, classical ML services, forecasting, ranking, vision, anomaly detection, and hybrid workflows, not only LLM-heavy tools.

If this layer is skipped, supplier shortlists become mixed-class comparisons, governance notes inherit the wrong obligations, and operating controls get attached to the wrong layer.
