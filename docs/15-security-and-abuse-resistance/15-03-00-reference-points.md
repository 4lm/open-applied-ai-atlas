# 15.3 Reference Points

_Page Type: Chapter Index | Maturity: Review-Ready_

This section gathers the named tools, standards, and review artifacts that make security decisions reusable. Use it when the threat boundary is already clear and the team now needs to compare enforcement options, map public frameworks into operating controls, and verify which evidence packet should exist before rollout or re-review.

## Included Reference Subsections

- 15.3.1 [Tools And Platforms](15-03-01-tools-and-platforms.md)
- 15.3.2 [Standards And Bodies](15-03-02-standards-and-bodies.md)
- 15.3.3 [Controls And Artifacts](15-03-03-controls-and-artifacts.md)

## What To Look For Here

- which runtime guardrails, policy engines, red-team utilities, provenance controls, or managed security platforms fit the system's actual trust boundary instead of acting as generic "AI security" badges
- which public standards and bodies should shape threat modeling, secure delivery, telemetry expectations, and supply-chain assurance without pretending they are interchangeable checklists
- which review artifacts should exist in one packet before rollout, including trust-boundary registers, tool permission manifests, abuse-test packs, provenance records, and containment drills

## Reading Boundaries

- Start with [15.3.1 Tools And Platforms](15-03-01-tools-and-platforms.md) when the control problem is known and the remaining question is which enforcement, testing, redaction, or provenance tooling fits the architecture and lock-in posture.
- Use [15.3.2 Standards And Bodies](15-03-02-standards-and-bodies.md) when reviewers need named external anchors for control design, threat vocabulary, or supply-chain assurance rather than a product comparison.
- Use [15.3.3 Controls And Artifacts](15-03-03-controls-and-artifacts.md) when the team needs a concrete review packet, release blocker list, or containment drill instead of another discussion about principles.
- Revisit [15.1 Security Foundations](15-01-00-security-foundations.md) or [15.2 Operating Security And Abuse Resistance](15-02-00-operating-security-and-abuse-resistance.md) if the argument is still about threat boundaries, approval design, incident evidence, or recurring operating cadence rather than named references.
